When AI Actually Belongs in Your Product (And When It Doesn't)

"Can you add AI to this?" is one of the most common requests founders bring to a build conversation. It's rarely the wrong instinct — but it's often the wrong question. The better question is: does this specific problem need a model that learns from data, or does it need a set of rules that a person could write down in an afternoon?

Start with the shape of the problem

Machine learning earns its complexity when a problem has patterns that are too varied, too subtle, or too numerous for a human to encode by hand — predicting churn from dozens of behavioral signals, ranking search results by relevance, or forecasting demand across thousands of SKUs. In those cases, a model can find structure a rules engine never would.

Plenty of features that get pitched as "AI," though, are really just conditional logic wearing a more exciting name. "Notify me when inventory drops below a threshold" is a rule. "Auto-categorize this expense based on the merchant name" can often be handled with a lookup table and a few pattern matches before a model is ever needed.

The real cost of choosing AI too early

A machine learning feature isn't just harder to build — it's harder to trust, harder to debug, and harder to explain to a customer when it gets something wrong. It also needs data to train on, which most early-stage products simply don't have yet. Shipping a rules-based version first often means shipping something usable in days instead of months, and it produces the very data you'd need to justify a model later.

A simple test before committing to AI

Before scoping a feature as an AI feature, it's worth asking three questions: Is there enough historical data to train on? Would a human expert struggle to write the rule by hand? And is the cost of an occasional wrong answer acceptable to the business? If the answer to any of these is no, a simpler system usually serves the product — and the budget — better.

Where AI genuinely changes the outcome

None of this is an argument against AI. Recommendation engines, document understanding, natural-language interfaces, and forecasting are all places where a model consistently outperforms hand-written logic, because the underlying pattern really is too complex to enumerate. The goal isn't to avoid AI — it's to reserve it for the problems that actually need it, so the investment pays for itself.
